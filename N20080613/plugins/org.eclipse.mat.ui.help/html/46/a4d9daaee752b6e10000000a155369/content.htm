FROM Clause

Specify the Class

The FROM clause defines the classes to be operated on. You have to specify the class by one of the following options:

●      Class name:

SELECT * FROM java.lang.String

●      A regular expression matching the class name:

SELECT * FROM "java\.lang\..*"

●      The object address of the class:

SELECT * FROM 0xe14a100

●      The object ID:

SELECT * FROM 3022

●      A sub select:

SELECT * FROM ( SELECT *

                FROM java.lang.Class c

                WHERE c implements org.eclipse.mat.snapshot.model.IClass )

The previous statement returns all objects in the heap. The implements check is necessary as the heap dump can contain java.lang.Class instances which originate from proxy classes.

The following query has the same effect and calls a method directly on the ISnapshot object:

SELECT * FROM $snapshot.getClasses()

Include Sub - Classes

Use the INSTANCEOF keyword to include objects of sub-classes in the query.

SELECT * FROM INSTANCEOF java.lang.ref.Reference

The resulting table contains, amongst other things, WeakReference and SoftReference objects because both classes extend from java.lang.ref.Reference. The following query has the same result:

SELECT * FROM $snapshot.getClassesByName("java.lang.ref.Reference", true)

Prevent Interpretation of the FROM Term as Classes

Use the OBJECTS keyword if you do not want to process the term as classes.

SELECT * FROM OBJECTS java.lang.String

The result is only the java.lang.String class object.

At the moment, the FROM OBJECTS term is experimental.
