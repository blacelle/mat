Class and Class Loader Histogram Views

Class Histogram View

To open a histogram view press the following toolbar button

The histogram shows the classes found in the snapshot, the number of objects for each class, the heap memory consumption of these objects, and, on demand, the minimum retained size of the objects. You can activate the computation using the context menu.

Retained Set/Size Definition

The retained set for a leading set of objects, such as all objects of a particular class or all objects of all classes loaded by a particular class loader or simply a bunch of arbitrary objects, is the set of objects which is freed if all objects of that leading set become unreachable. The retained set includes these objects as well as all other objects only reachable through these objects. The retained size is the total heap size of all objects contained in the retained set.

The Minimum Retained Size is an estimate of the retained size which is calculated much faster than the real retained size of a set of objects.

The Minimum Retained Size almost always either matches the precise retained size or is very close to it. It only depends on the number of objects in the inspected set, not the number of objects in the heap dump. It is about 1,000 times faster than computing the precise retained size.

Class Loader Histogram View

To switch between the Class and Class Loader histogram, choose the following button:

The class loader view shows you the same information as the class view, but the classes are grouped by the class loaders that load and define the various classes.

Class loaders also mean deployment unit in the context of the SAP application server. Therefore, if possible, human-readable names are displayed in this view matching to a certain extent the names of the Software Development Archives (SDAs). The retained size on a class loader is the amount of memory that is garbage collected if all objects of all classes of the class loader are no longer reachable by the GC.

You can search for certain classes or class loaders by entering a regular search expression in the text entry field and choosing the Find Now button.

Try out the context menus, they give you more functionality. This refers to all views. The context menus are context-sensitive, so for classes you see other menus as for objects. Certain objects like HashMap have additional functionality, such as showing you the keys/values of the data structure.
