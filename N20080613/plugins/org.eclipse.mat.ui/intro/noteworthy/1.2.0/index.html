New and Noteworthy

Release 0.7.0

Overview Page The new Overview page provides a quick look at the size of the heap dump, the biggest objects and most frequently used actions.

Leak Suspect Report The leak suspect report lists possible memory leaks.

The report lists
  • big objects or group of objects and
  • threads consuming excessive processing space.

Please note the keywords available to search for known problems and the CSN component (if available) to forward the problem to the right development organization.

The small ZIP file containing HTML can easily be send via email. No transfer of multi GB heap dumps necessary.
Drill Down Drill down to list
  • the accumulation point (who is keeping the memory) and
  • the retained set of the accumulation point (what is kept alive)
If the report is viewed inside the Memory Analyzer, use the hyper-links to view the objects.

Run Report Either select the report from the tool bar...

... or execute the command line script: 
ScriptParseHeapDump -report default:support.xml <path to hprof>
... or mail the result to an email address: 
ScriptParseHeapDump -report default:support.xml -mail <recepients> -smtp_gateway <host> -smtp_from <address> <path to hprof>
Java Locals Objects marked as Garbage Collection Roots of type "Java Local" are now referenced from the thread.

For example, now it is very easy to see which objects the Finalizer thread currently processes.
Threads List threads plus details with Java Basics -> Thread Overview.

Then drill down into the details.

If the thread is currently processing a request, more details are displayed. Known requests are HTTP, JCO, JDBC, socket input streams.

Usability Improvements

Search Just start typing to highlight elements.

Filter Filtering is now available in all views. For example, one can filter tree to see only children greater than a limit.

Grouping We have added more grouping options.
