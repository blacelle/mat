New and Noteworthy

Older Releases ( < 0.6.0)

Paths from GC roots Exclude weak and soft references in Paths from GC roots. Also, check out the paths in the console. The command can now exclude arbitrary class/field combinations.

Paths from GC roots Now shows for thread-bound GC roots (JNI locals, Java locals, busy monitors) also the thread itself.

Calculate retained set on pattern Calculate the retained set on all objects whose classes match a regular expression.

Notes For your convenience, you can jot down notes with every heap dump. The notes are saved in a file named <heap dump name>.notes.txt.

Object Query Language The Object Query Language (OQL) let's you perform custom SQL-like queries on the heap dump. Just think of classes as tables, objects as rows and fields as columns.

Support for 6.40/7.00 The sessions view and command, the list_loaders command, and the leaking_loaders command are extended to work with heap dumps of 6.40/7.00 systems.

Usability Improvements

Object Icons To easily identify objects, classes, arrays and class loaders have their own icons. Any garbage collection root is decorated with a orange dot in the lower left corner. Press Alt-I for help on icons.

Object Inspector The new inspector is now available in all views and fully supports all possible context actions.

Customize Layout Arrange available windows as you see fit. The layout will be restored when you start the Analyzer the next time.

Window Management Right click on the window title to manage the open views on your heap dump.
