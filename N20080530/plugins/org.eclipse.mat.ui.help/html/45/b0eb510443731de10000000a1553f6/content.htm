Memory Analyzer

Overview

The Memory Analyzer is a tool for analyzing plain heap dumps. It can be used as a health-check or for post-mortem analysis. With the Memory Analyzer you can look into the Java VM by analyzing the heap dumps (HPROF) written by Sun JVMs (1.4.2_12 or higher and 1.5.0_07 or higher), HP-UX VM (1.4.2_11 or higher) and SAP JVMs (since 1.5.0).

Memory Analyzer Functions

The Memory Analyzer runs on snapshots that are standard HPROF heap dumps.

A heap dump is a dump of the Java heap and some perm space class info, with all objects and their references among the objects. The heap dump can be activated using a VM parameter. For more information see Acquiring a Heap Dump

JVM is stopped during the time the heap dump is written. 

To open a saved snapshot, choose the following button:

When you open a heap dump for the first time, it is parsed. Parsing is a process that transforms a heap dump into a Memory Analyzer snapshot for faster access. The snapshot is opened right after parsing and the Memory Analyzer shows the following views:

●      Snapshot Details View – Gives information about the currently active snapshot.

●      Bookmarks View – Allows you to create bookmarks for all interesting objects and reopen them later to continue your analysis. The view lists current bookmarks and allows operations on them via a context menu.

●      Object Inspector View – Provides full information about the selected object and supports various context actions.

●      Notes View – Allows you to write down notes with every heap dump. The notes are saved together with the heap dump.

               

If you work on bigger heap dumps, you have to switch to a 64 bit machine or you need someone else who can do the parsing of such huge heap dumps on a 64 bit machine for you. During parsing, index files are created. Together with the heap dump, these index files form the snapshot. If all files of the snapshot are copied to another machine, a Memory Analyzer there can reuse them without re-parsing the heap dump. Even enormous heap dumps can usually be analyzed on 32 bit machines once they have been parsed on a 64 bit machine.

Overview of General Features

In general, the functionality for objects is available as a context menu for objects and the functionality for classes is available as a context menu for classes. Sessions are also objects, so they have at least the context menu entries for objects and some other specific entries.

To copy the content of the first selected cell into the clipboard, press CTRL+C or use more export features:

·        To export a class or class loader histogram to a CSV file, select the  button.

·        To export the currently displayed view as text, including the expanded or collapsed nodes as they are shown in the view itself, select the   button.
