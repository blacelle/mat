Property Accessors

Accessing Fields of the Heap Object

Properties of heap objects are accessed using a simple dot notation:

[ <alias> . ] <field> . <field> . <field>

An alias can be defined in the FROM clause to identify the current object, for example a row in the SQL analogy in which the OQL statement operates. Without an alias, the field is assumed to be one of the fields of the current object.

Fields are attributes of the Java objects in the heap dump. You have to use the Object Inspector to find out the available fields of an object.

Accessing JavaBeans Properties

As the original objects in the heap dump are inactive and you cannot access their attributes, the Memory Analyzer creates parallel objects that represent the original objects, such as IObject,IClass and IArray. Using the @ symbol, OQL accesses attributes of the underlying Java objects. The attributes are resolved via Bean Introspection.

[ <alias> . ] @<attribute> ...

Calling Java Methods

Adding parentheses forces OQL to interpret this as a Java method call. The call is executed via reflection.

[ <alias> . ] @<method>( [ < expression >, < expression > ] ) ...
