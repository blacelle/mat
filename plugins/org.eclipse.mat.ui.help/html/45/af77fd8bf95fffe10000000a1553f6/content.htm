Dominator Tree View

If you refer to the memory footprint analysis, the Dominator Tree view is one of the most helpful views. It is based on the same semantics as the retained set/size calculation. The Dominator Tree view represents:

●      The dominated objects

For a particular object, which other objects depend on it and are garbage collected if the particular object becomes unreachable.

●      The object’s dominator chain up to the root

For a particular object, which objects keep the particular object from being garbage collected.

Dominator Tree Grouping

The Dominator Tree can be grouped by classes, classloaders or packages. If no single huge objects can be seen in the Dominator Tree, grouping helps to identify classes, classloaders or packages that occupy a lot of memory.

Definition of Dominator and Dominator Tree

A node d dominates on a node n if all the paths from the start node to node n pass through node d. The transformation of the object reference graph into a dominator tree gives you the possibility to easily identify the biggest chunks of retained memory.

Immediate Dominators

Immediate dominators are calculated for groups of objects. Immediate dominators are the direct parents of the investigated objects in the dominator tree grouped by class.
