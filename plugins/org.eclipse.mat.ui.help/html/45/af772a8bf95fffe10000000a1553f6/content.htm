Paths to GC Roots View

The Paths to GC Roots view analyzes which objects are responsible for keeping a single object in the heap.

You can reach this view using context menu Path to GC Roots.

The Paths from GC Roots view shows the first group of paths from GC Roots to the object of interest. The shortest paths are shown first and more can be requested on demand until no more paths are found.

The view comes in the form of a tree. The paths to sub-objects that are objects below the original object of interest are not necessarily the only references to those objects, as only the shortest paths are displayed. That is, if more than one path leads from one GC Root to the original object of interest, only the shortest path is displayed. To learn more about the paths of a sub object from the GC Roots, you have to open the view to the object that is of interest. This guarantees that the result is correct; otherwise there may be missing paths.
